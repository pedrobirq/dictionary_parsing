# GUI

## Описание проекта
Инструмент для автоматизации парсинга статей с графическим интерфейсом (GUI) на базе `tkinter`. Проект разделен на универсальный модуль интерфейса и конкретную реализацию логики обработки.

## Структура файлов
- `gui.py`: Содержит класс `ParsingApp`, который реализует GUI. Он абстрагирован от логики обработки файлов и принимает функцию обработки как аргумент.
- `gui-example.py`: Пример использования `ParsingApp` с функцией-заглушкой для демонстрации работы интерфейса.

## Функционал GUI (`gui.py`)
Интерфейс предоставляет следующие возможности:
- **Выбор директорий**: Выбор папки с исходными статьями и папки для сохранения результатов.
- **Управление процессом**: 
    - Кнопка **Start/Stop**: Запуск процесса парсинга всех файлов в выбранной папке или его принудительная остановка.
    - **Progressbar**: Визуальное отображение прогресса обработки файлов.
- **Логирование**:
    - Окно вывода логов в реальном времени с метками времени.
    - Кнопка **Очистить**: Очистка окна логов.
    - Кнопка **Сохранить**: Сохранение содержимого лога в текстовый файл.
- **Многопоточность**: Работа парсера вынесена в отдельный поток, чтобы интерфейс оставался отзывчивым.

## Как использовать (`gui-example.py`)
Для запуска демонстрационного режима выполните:
```bash
python gui-example.py
```
В примере используется функция `process_article`, которая имитирует обработку файла (ожидание 1 секунда) и случайным образом генерирует ошибки для проверки обработки исключений в GUI.

## Интеграция собственной логики
Чтобы использовать GUI для своих задач:
1. Импортируйте `ParsingApp` из `gui.py`.
2. Создайте функцию обработки с сигнатурой:
   `def my_process_func(source_path: str, target_path: str, logger_func: Callable[[str], None])`
3. Передайте эту функцию в конструктор `ParsingApp(root, my_process_func)`.
